# House-keeping-requests-Analysis-
This project is about analysis and system optimization of infrastructure services 
# Модуль автоматизации распределения аварийных заявок ЖКХ

## Описание проекта/project description 
Данный пет-проект посвящен проектированию ИТ-системы для автоматического распределения экстренных заявок между мастерами управляющей компании (УК). 
**Цель:** Сократить время назначения мастера на аварийный участок с 40 минут до 2 минут за счет исключения ручного поиска диспетчером.

## Бизнес-ценность и доменная экспертиза
Проект опирается на мой личный практический опыт в сфере ЖКХ (Мастер 4 разряда). Система решает реальные боли отрасли: неоптимальную логистику бригад, человеческий фактор при выборе исполнителя и задержки в передаче данных.

## Структура проекта/project structure 
* `docs/use_cases.md` — Подробные сценарии использования системы.
* `diagrams/` — BPMN-схемы процессов As-Is / To-Be и ER-диаграмма базы данных.
* `api/assign_request.json` — Пример структуры интеграционного JSON-пакета.
* `db/search_master.sql` — SQL-скрипт для алгоритма подбора исполнителей.

## Стек и инструменты/tools
* **Моделирование процессов:** BPMN 2.0 (Miro/Camunda)
* **Работа с данными:** PostgreSQL, JSON
* **Проектирование API:** REST API
* **Интеграции:** ГИС ЖКХ (SOAP/REST по ФЗ №209)
